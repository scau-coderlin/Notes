# 免密设置

```sh
git config --global credential.helper store #设置记住密码
git config --global user.name "xxx" #设置账号名
git config --global user.email "xxx@keyou.cn" #设置邮箱
```



# 本地仓库上传git

本地已有文件夹，并将本地文件夹的内容上传至git仓库。

## 远程

创建新仓库：

gitee 使用右上角+号创建即可，



## 本地

```shell
cd dir

# 初始化
git init

# 查看文件状态, 红色表示未添加到仓库
git status

# 将所有文件添加到仓库
git add *

# 查看文件状态, 绿色表示已添加到仓库
git status

# 将项目文件夹提交到 Git 仓库中, -m 后面的内容是本次提交时所带的注释内容。
git commit -m "first commit"

# 关联到远端
git remote add origin [https://gitee.com/scau_coderlin/Notes.git]

# 推送
git push -u origin "master"

#### err
# 如果报错信息如下:
$ git push -u origin "master"
remote: [session-ae48fbba] Access denied
fatal: unable to access 'https://gitee.com/scau_coderlin/Notes.git/': The requested URL returned error: 403
# 解决方法:
# 1. 查看配置文件的用户凭证
cat ~/.gitconfig 
# windows
# 2. 对比windows存储的默认账户命名, 路径: 控制面板->所有控制面板项->凭据管理器
# 3. 修改至一致, 可删除windows里的凭证
# linux
rm -rf 
```



# 新建分支

- 克隆分支：

```ch
git clone xxx.git
```

- 在本地创建分支：

```sh
git branch xxx git_hash # xxx为分支名, git_hash 可省略
```

- 将本地分支推送到远程仓库：

```sh
git push --set-upstream origin xxx # xxx为分支名
```

# 合并

- 查看本地分支：

  ```shell
  git branch
  ```

- 将本地分支切换到 `master`：

  ```shell
  git checkout master
  ```

- 查看是否切换成功：

  ```shell
  git branch
  ```

- 拉取主分支代码：

  ```shell
  git pull
  ```

- 切换到开发分支：

  ```shell
  git checkout linj
  ```

- 将主分支合并到开发分支：

  ```shell
  git merge --no-ff master
  ```

- 在本地解决冲突，解决冲突后，重新正常上传一遍即可。

- 在 `git` 网站上创建合并请求。

# 获取当前提交的hash

```shell
git log --pretty=format:%h -1

# 切换到某次提交
git checkout hash
```

# 版本回退

```sh
git reset --hard git_hash
git push -f
```
